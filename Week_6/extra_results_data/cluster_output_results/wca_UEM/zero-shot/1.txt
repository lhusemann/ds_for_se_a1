This code provides a terms index writer for Lucene, which is used to index terms and their associated metadata for efficient searching. It uses an FST (Finite State Transducer) to store the index terms in a prefix trie, and provides a pluggable policy for selecting index terms. The index terms are sorted using unsigned byte term sort order, which is equivalent to unicode codepoint order when the bytes are UTF8. The code also includes a terms dictionary reader, which reads the terms dictionary from a file and populates a TreeMap, where the keys are field names and the values are TermsReader objects. Each TermsReader object represents a field and contains information about the number of terms, total term frequency, document frequency, and document count for the field. The code also provides a custom implementation of the PostingsFormat class for Lucene, which uses the OrdsBlockTreeTermsWriter and Lucene90PostingsWriter classes to write and read postings data for Lucene indexes.
