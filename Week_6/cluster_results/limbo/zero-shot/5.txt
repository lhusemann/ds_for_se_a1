Title: BloomFilteringPostingsFormat - A Lucene PostingsFormat Implementation Using Bloom Filters

Description:
The BloomFilteringPostingsFormat is a Lucene PostingsFormat implementation that uses Bloom filters to efficiently determine if a given element is present in a set. It maintains a set of Bloom filters for a selection of fields created in the index and records them as a segment summary in an additional "blm" file. The format of the blm file includes a header, delegate postings format name, number of filtered fields, filter<sup>NumFilteredFields</sup>, and a footer. The code provides a choice of delegate PostingsFormat and a BloomFilterFactory to customize the functionality of the Bloom filter. The default configuration uses a DefaultBloomFilterFactory which allocates a ~8mb bitset and hashes values using MurmurHash2. The code also includes a BloomFilteredFieldsProducer class that reads the Bloom filters from the blm file and provides a filtered view of the terms in the index. The BloomFilteredFieldsConsumer class is responsible for writing the Bloom filters to the blm file during the indexing process.
