**Title:** Lucene Codec
**Summary:** The Lucene Codec module is responsible for encoding and decoding inverted index segments. It provides methods for encoding and decoding postings, docvalues, stored fields, term vectors, field infos, segment info, document normalization values, live docs, compound files, points index, and numeric vector fields. It uses the Service Provider Interface (SPI) to resolve codec names and provides methods for looking up a codec by name, getting the list of available codec names, reloading the codec list from a given classloader, and getting the default codec used for newly created IndexWriterConfig objects. The module is abstract and requires subclasses to implement the abstract methods for each encoding/decoding functionality. The subclasses must also provide a no-arg constructor so that the SPI can load them. The module also provides a static inner class called Holder, which delays the initialization of default codecs and available codecs until needed. This prevents classloading deadlock. The module provides a toString method that returns the codec's name.
