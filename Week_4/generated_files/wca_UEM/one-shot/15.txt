**Title:** Postings API — Abstract API for Producing Terms, Doc, Freq, Prox, Offset, and Payloads Postings

**Summary:**
The Postings API is an abstract API that produces terms, doc, freq, prox, offset, and payloads postings. It is composed of two layers: a `Fields` interface for accessing terms, doc, freq, prox, offset, and payloads postings, and a `FieldsProducer` abstract class that extends `Fields` and provides methods for accessing and iterating over terms, doc, freq, prox, offset, and payloads postings. The `FieldsProducer` class also includes methods for closing the producer and checking its integrity.

The architecture follows a clean separation of concerns, with the `Fields` interface exposing a high-level API for accessing postings, and the `FieldsProducer` abstract class exposing a lower-level API for producing postings. The `FieldsProducer` class is responsible for implementing the core logic of producing terms, doc, freq, prox, offset, and payloads postings, while the `Fields` interface acts as a facade that hides the implementation details.

The Postings API is designed to be consumed by higher-level components that need to access and iterate over terms, doc, freq, prox, offset, and payloads postings. It isolates the core logic of producing postings from the rest of the system, allowing for easy testing and swapping of different implementations.

The Postings API is a leaf cluster with minimal coupling to the broader codebase, as it only depends on the standard Java Collections API. It isolates the core logic of producing postings, making it a self-contained module that can be easily maintained and extended.
