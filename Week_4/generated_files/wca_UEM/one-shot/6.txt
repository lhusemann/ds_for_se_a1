**Title:** CompoundFileReader — Read-Only Directory Implementation for Lucene Compound Files

**Summary:**
The CompoundFileReader module provides a read-only Directory implementation that is optimized for reading data from Lucene compound files. It is a low-level data access primitive that isolates the compound file format from the higher-level Lucene index and document models.

The CompoundFileReader class implements the abstract methods of the Directory class, providing the necessary functionality to open, read, and close files within the compound file. It uses a compound file reader implementation to handle the low-level details of reading from the compound file, while exposing a read-only view of the directory structure to the caller.

The module follows a clear separation of concerns, with the compound file reader implementation handling the low-level details of reading from the compound file and the CompoundFileReader class exposing a read-only view of the directory structure to the caller. It is designed to be used in conjunction with the Lucene index and document models, providing a seamless integration with the rest of the Lucene ecosystem.
