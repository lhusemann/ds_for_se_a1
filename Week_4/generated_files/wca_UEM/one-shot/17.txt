**Title:** HashFunction — Abstract Base Class for Hashing Implementations

**Summary:**
The `HashFunction` class is an abstract base class that defines a thread-safe interface for implementing a hash function. It provides a single method `hash` that takes a `BytesRef` object as input and returns an integer hash value. The implementation of the `hash` method is expected to be thread-safe and provide a consistent hash value for the range of bytes referenced by `bytes.offset` and `bytes.length`.

The `HashFunction` class is part of the `org.apache.lucene.codecs.bloom` package, which provides utilities for working with Bloom filters, a space-efficient data structure for approximate set membership tests. The `HashFunction` class is designed to be subclassed by concrete implementations of hash functions, such as `MurmurHash2` or ` papHash`, which can then be used to generate hash values for byte sequences.

The `HashFunction` class itself has no state and does not maintain any instance variables. All state is held in the input `BytesRef` object, which is passed to the `hash` method. This makes the class thread-safe and ensures that each invocation of the `hash` method produces a consistent result based on the contents of the input `BytesRef`.

The `HashFunction` class has no external dependencies beyond the `BytesRef` type and the Lucene utility classes it is part of. It is designed to be a standalone module that can be easily extended and subclassed to provide different hash function implementations.
