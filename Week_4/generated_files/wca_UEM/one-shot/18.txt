**Title:** KnnVectorsFormat — Per-Document Vectors Format

**Summary:**
The KnnVectorsFormat module provides functionality for encoding and decoding per-document vectors, as well as associated indexing structures, required for nearest-neighbor search. It consists of three main components:

- `KnnVectorsWriter`: An abstract class that defines the interface for writing vectors to the index. It is responsible for writing vectors to the index in a format that can be later retrieved and used for nearest-neighbor search.

- `KnnVectorsReader`: An abstract class that defines the interface for reading vectors from the index. It is responsible for reading vectors from the index and providing access to the encoded vector data for use in nearest-neighbor search.

- `KnnVectorsFormat`: A class that provides methods for creating instances of `KnnVectorsWriter` and `KnnVectorsReader` based on the specified format. It is responsible for managing the lifecycle of the writer and reader instances and ensuring that they are properly closed when they are no longer needed.

The module is designed to be pluggable, allowing different formats to be supported by implementing the `KnnVectorsWriter` and `KnnVectorsReader` interfaces and registering them with the `KnnVectorsFormat` class. This allows users to choose the format that best suits their needs for encoding and decoding vectors.

The code is part of the Lucene library, which is a widely-used open-source search engine library. It provides efficient and scalable indexing and searching capabilities for large-scale data sets, and the KnnVectorsFormat module is a key component of the library's support for nearest-neighbor search.
